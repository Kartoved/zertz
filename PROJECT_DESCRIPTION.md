# ZERTZ (Web) — Описание проекта

## 📌 Обзор
Браузерная реализация абстрактной стратегической настольной игры ZÈRTZ.
Проект поддерживает локальный режим (за одним устройством) и онлайн-мультиплеер в реальном времени (а также заочные партии).

### 🛠 Технологический стек:
- **Фронтенд:** React 18 + TypeScript + Vite
- **Управление состоянием:** Zustand (отдельные сторы: `gameStore` для локальной игры, `roomStore` для онлайн-партий, `authStore` для авторизации)
- **Стилизация:** TailwindCSS (включая тёмную/светлую темы)
- **Хранение данных:** IndexedDB (локально) + PostgreSQL (на сервере)
- **Бэкенд:** Express.js (Node.js) с драйвером `pg` для БД
- **Маршрутизация:** React Router v6
- **Локализация:** Встроенная кастомная система i18n (поддерживает русский, английский и эсперанто языки)

## 🎲 Реализованные правила игры

### Структура хода:
1. **РРазмещение (если нет обязательных взятий):**
   - Игрок выбирает цвет шарика (белый/серый/черный) из запаса (или из своих захваченных, если запас пуст).
   - Размещает его на любом свободном кольце.
   - Удаляет одно «свободное» кольцо с края доски (обязательно, если такое есть).
   Кольцо считается свободным, если на нём нет шарика, и оно имеет не менее 2 открытых внешних граней.
   - Ход переходит к сопернику.

2. **Взятие (обязательно при наличии возможности):**
   - Шарик перепрыгивает через соседний шарик любого цвета на пустое кольцо за ним (как в шашках).
   - Шарик, через который перепрыгнули, считается захваченным (забирается в пул игрока).
   - Множественные взятия: если после прыжка шарик может взять еще один — он **обязан** продолжить серию взятий.
   - Как только серия окончена, ход завершен.

### Изоляция колец:
- Удаление кольца может разделить доску на несколько изолированных групп.
- Если оторвавшаяся группа колец полностью заполнена шариками (нет ни одного пустого кольца), все шарики в этой группе автоматически захватываются игроком, который совершил это действие.

### Условия победы (достичь любого из них первым):
- Набрать **4 белых** шарика
- Набрать **5 серых** шариков
- Набрать **6 чёрных** шариков
- Набрать по **3 шарика каждого цвета**

**Размеры полей:** 37 / 48 / 61 колец (выбирается при создании новой игры).

## 👤 Аутентификация и профили

**Регистрация и Вход:**
- Аутентификация по никнейму и паролю. Пароли шифруются с помощью bcrypt (12 rounds).
- Используется JWT токен (сохраняется в localStorage, срок действия 30 дней) для автологина.

**Профили:**
- В профиле отображается: ник, рейтинг Glicko-2, дата регистрации, статистика побед/поражений, винрейт, лучшая серия побед.
- Возможность редактирования цитаты, установки контактной ссылки и изменения страны/флага.
- Имеется система друзей (подписок) на других игроков.

**Рейтинг Glicko-2:**
- Рейтинговая система рассчитывает силу игроков. Начальный рейтинг — 1500.
- База рейтинга пересчитывается после ранговых матчей онлайн.

**Глобальный рейтинг (Лидерборд):**
- Доступна таблица со всеми игроками, с сортировкой по победам, винрейту, количеству игр и самому рейтингу.

## 🌐 Онлайн мультиплеер и чат

1. Игрок создаёт игру («Сыграть по ссылке» или авто-матчмейкинг), выбирает контроль времени (например Рапид, Блиц или Заочная).
2. Выдается ID комнаты и ссылка для друга (или комната ожидает оппонента в лобби).
3. При подключении синхронизируется стейт через API (`roomsApi.ts`).
4. В онлайн-игре работает чат (`ChatPanel.tsx`), там же отображаются системные сообщения (Запрос на отмену хода, Принятие, Отклонение).
5. Игроки могут **Сдаться** или **Предложить ничью/Отменить ход**.
6. Состояние игры регулярно поллится с бэка, обеспечивая надежность до перехода на WebSockets.

## 🏗 Архитектура проекта (Кратко)

- `src/game/` — Локальная логика движка, геометрия гексов, проверка победы и генерация ходов (Engine).
- `src/store/` — Состояние приложения.
  - `gameStore.ts` — для локальной игры (на одном экране).
  - `roomStore.ts` — для мультиплеера онлайн.
  - `authStore.ts` — токен, сессия текущего юзера.
- `src/components/Board/` — SVG-рендер доски, колец и шариков, зуминг.
- `src/components/Online/` — Окно онлайн-комнаты, чатов, профиля оппонента.
- `server/` — Бэкенд (Node.js/Express) + логика PostgreSQL.

## 🚀 Деплоймент
Продукт легко деплоится на Railway:
- Сборка: `npm run build` (компилирует статику Vite в папку `dist/`).
- Старт бэкенда: `npm start` (поднимает Express сервер, который раздаёт API и статику).
- Необходимы переменные окружения: `DATABASE_URL`, `JWT_SECRET`.
