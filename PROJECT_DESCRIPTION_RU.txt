ZERTZ (Web)

Обзор
- Браузерная реализация настольной игры ZERTZ.
- Режимы: локальный (hot-seat) и онлайн (мультиплеер через сервер).
- Стек: React + TypeScript + Vite, Zustand (gameStore для локальных, roomStore для онлайн), IndexedDB + PostgreSQL для сохранения.
- Бэкенд: Express.js (Node) + pg.
- Отрисовка: SVG-доска, flat-top ориентация, axial-координаты, размеры досок 37/48/61.

Реализованные правила
- На ходу игрок выполняет одно из двух действий:
  1) Выставление:
     - Выбрать цвет шарика из резерва.
     - Поставить на пустое кольцо.
     - Удалить одно доступное кольцо (если есть) и ход заканчивается.
  2) Взятие (обязательно):
     - Если взятие возможно, игрок обязан бить.
     - Взятие как в шашках: прыжок через соседний шарик на пустое кольцо сразу за ним.
     - Цепочка взятий обязательна: если после прыжка можно бить дальше, ход продолжается до конца.

Правила удаления колец
- Кольцо можно удалить только если:
  - Оно пустое.
  - У него есть минимум две соседних свободных стороны (признак края).
- Изоляция разрешена: удаление может разорвать доску.
  - Если отделившаяся группа колец не содержит пустых колец (только шарики), все шарики в группе автоматически засчитываются как взятые текущим игроком.

Журнал ходов / Варианты
- Ходы хранятся в виде дерева (основная линия + альтернативные ветки).
- При переходе на ход состояние полностью восстанавливается путём проигрывания ходов от начала.
- Ветки отображаются сразу после хода-родителя (в скобках).
- Навигация:
  - Клик по ходу.
  - Клавиши ArrowLeft / ArrowRight.

Автосохранение / Загрузка
- Игра автоматически сохраняется в IndexedDB + PostgreSQL после каждого хода.
- Онлайн-партии также сохраняются локально и доступны для загрузки.
- В главном меню есть "Загрузить игру" со списком всех сохранённых партий (завершённых и незавершённых).

Авторизация / Профили
- Регистрация и вход по нику и паролю (bcrypt хеширование, 12 раундов с солью).
- JWT-токен хранится в localStorage (30 дней), автоматическая авторизация при загрузке.
- Валидация: ник 2-24 символа, пароль мин. 8 символов с обязательным спецсимволом.
- Профиль: ник, рейтинг Глико-2, статистика побед/поражений, лучшая серия побед, цитата, флаг страны.
- В главном меню кнопка "Войти / Зарегистрироваться" или "Профиль" (если залогинен).
- В онлайн-играх ник залогиненного пользователя подставляется автоматически, редактирование запрещено.
- В локальных играх ник залогиненного пользователя подставляется как Игрок 1 автоматически.

Рейтинг / Рейтинговые игры
- Система рейтинга Глико-2 (начальный: 1500, RD 350, vol 0.06).
- При создании онлайн-игры авторизованный пользователь может включить тумблер "Рейтинговая игра".
- По завершении рейтинговой игры рейтинги обоих игроков пересчитываются по Глико-2.
- Таблица игроков доступна из главного меню (кнопка "Игроки") с сортировкой по столбцам.

UI / UX
- Кольца круглые и видны даже когда на них стоит шарик.
- Зум поля колесом мыши (камера-зум через SVG viewBox).
- Звуки на основные действия (постановка, удаление, взятие, победа, отмена).

Локальная разработка
- Установить зависимости и запустить:
  - npm install
  - npm run dev

Релиз (чтобы друг запустил без установок)
- Статическая сборка лежит в папке release/.
- Запуск на Windows:
  - Двойной клик release/Start.cmd (или запуск Start.ps1)
  - Открыть в браузере http://localhost:5173

Структура проекта (кратко)
- src/game/*: игровая логика/движок и геометрия доски.
- src/store/*: Zustand store.
- src/components/*: UI.
- src/db/*: IndexedDB утилиты.
- release/*: готовая сборка и небольшой PowerShell сервер.
