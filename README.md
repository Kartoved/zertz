# ZERTZ (Web)

Браузерная реализация настольной игры ZERTZ с поддержкой локального (hot-seat) и онлайн (мультиплеер через сервер) режимов.

> ⚠️ ZERTZ — это абстрактная стратегическая настольная игра серии GIPF Project. Сама игра и её правила являются собственностью **Kris Burm**. Этот проект — fan-реализация программного клиента; он распространяется под AGPL-3.0 и не претендует на права на саму игру.

## 🛠 Технологический стек
- **Frontend:** React, TypeScript, Vite, TailwindCSS
- **State Management:** Zustand (`gameStore` для локальных игр, `roomStore` для онлайн)
- **База данных:** IndexedDB (ранее), PostgreSQL (бекенд)
- **Backend:** Express.js (Node), pg (PostgreSQL)
- **Рендеринг:** SVG доска с координатной сеткой (axial coordinates)

## 🗺️ Игровые механики
Игра полностью реализует правила оригинального ZERTZ (включая постановку, взятие, изоляции и проверку свободных колец).
Размеры досок: 37, 48 и 61 кольцо.

Поверх этого: рейтинг Glicko-2, лобби и матчмейкинг, вызовы, глобальный чат, дебют-эксплорер, correspondence-партии с условными пре-мувами, ZERTZ TV, **Студии** (Lichess-style уроки с режимом тренировки) и своя нотация **ZEN / ZIP** (аналоги PGN/FEN — копировать/экспортировать/импортировать позиции и партии).

## 🚀 Запуск проекта
Локальная разработка (Postgres в Docker, фронт + бэк в двух терминалах):
```bash
docker compose up -d db   # postgres на 5432 (только db, не весь стек)
cp .env.example .env      # впиши JWT_SECRET
npm install
npm run dev               # Vite на 5173 (проксирует /api → 5050)
npm run dev:server        # Express на 5050
```

Сборка в production:
```bash
npm run build             # tsc + vite build → dist/
```

Деплой (Docker Compose на VDS) и API-эндпоинты — в [DEPLOY.md](./DEPLOY.md). Полное описание подсистем — в [CLAUDE.md](./CLAUDE.md) и [PROJECT_DESCRIPTION.md](./PROJECT_DESCRIPTION.md).
