# 💡 Идеи по улучшению проекта ZERTZ (Web)

Статус на май 2026. ✅ = реализовано, 🔶 = частично, ❌ = не реализовано.

---

## 🎮 Геймплей и Механики

1. ✅ **Игра против ИИ (Умный бот)**
   - Minimax с альфа-бета отсечением реализован (`src/ai/`), работает в Web Worker.
   - UI кнопка пока скрыта («coming soon») — логика готова, нужна только финальная полировка.

2. ❌ **Подсказки для новичков (Tutorial)**
   - Не реализовано. Идея: подсветка обязательных взятий или результативных ходов при первых партиях.

3. 🔶 **Анализ завершённых партий**
   - Analysis mode работает для **текущих** онлайн-игр (correspondence), с пре-мувами и навигацией по дереву.
   - Анализ **завершённых** партий из истории профиля — не реализован.

4. ❌ **Вариации правил**
   - Нестандартные расстановки или кол-во шариков — не реализовано.

---

## 🖥 Пользовательский интерфейс (UI / UX)

5. 🔶 **Звуковое оформление**
   - Звуки ходов (размещение шарика, снятие кольца, взятие, победа) реализованы.
   - Не реализовано: звук входящего сообщения в чате, предупреждение за 10с в блице, регулировка громкости.

6. ❌ **Анимации движения шариков**
   - CSS/SVG анимации прыжка при взятии и «отвала» при изоляции — не реализовано.

7. ❌ **Аватары игроков**
   - Пока инициал в цветном круге. DiceBear или загрузка своего изображения — не реализовано.

8. ❌ **Эмодзи-реакции в игре**
   - Быстрые реакции (👍 😢 😠 🤝) вместо/рядом с чатом — не реализовано.

---

## 👥 Социальные и игровые функции

9. 🔶 **Обсервер-режим (Зрители)**
   - Базовый режим есть: `?watch=1` открывает партию без возможности ходить.
   - Нет: список активных игр для зрителей, специальный UI наблюдателя.

10. ❌ **Режим турниров (Сетки)**
    - Швейцарская/олимпийская система, организация внутри сайта — не реализовано.

11. ❌ **Система Ачивок (Достижений)**
    - Бейджи в профиль за первую кровь, изоляцию, серию побед и т.д. — не реализовано.

12. ✅ **Оповещения о заочных ходах**
    - Web Push Notifications полностью реализованы (VAPID, сервер рассылает при ходе оппонента).
    - Email через mailer.js (magic-link аутентификация).

## ⚙ Технические улучшения

13. 🔶 **Переход на WebSockets**
    - Остаётся на HTTP polling, но сильно оптимизировано: HEAD-эндпоинт `/head` делает тяжёлый fetch только при реальном изменении (~90% запросов в покое стали лёгкими).
    - WebSockets дадут ещё меньше задержки, но текущее решение вполне приемлемо.

14. ✅ **Поддержка PWA (Progressive Web App)**
    - Service Worker, Web App Manifest, `vite-plugin-pwa` — реализовано. Игра устанавливается на смартфон.

15. 🔶 **Мульти-локализация**
    - Реализовано 3 языка: русский, английский, эсперанто.
    - Испанский, французский, немецкий — не добавлены.

## 🆕 Реализовано сверх плана (не было в IDEAS.md)

- **Лобби** — публичные слоты с TTL, создание/вступление без предварительного знакомства

- **Opening Explorer** — база дебютов из сыгранных партий, drill-down по позиции
- **Correspondence pre-moves** — условные пре-мувы для заочных игр с авто-исполнением
- **Рейтинговая система** — Glicko-2, отдельная статистика рейтинговых партий
- **Вызовы / инвайты** — отправка приглашения конкретному игроку
- **Глобальный чат** — с пагинацией, системными сообщениями, анонсами новостей
- **Blog / News** — markdown-посты с локализацией (ru/en/eo), анонсы в чат
- **Профили игроков** — рейтинг, история партий, подписки, страна, цитата
- **Таблица лидеров** — топ по рейтингу с фильтрацией
- **Безопасность v0.14.0** — серверная верификация state, auth на всех мутирующих роутах, rate limiting
